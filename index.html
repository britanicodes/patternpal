Problem And Baseline Workflow

Garment fitting is the highest-friction step in home sewing, especially for beginners.

  1. Sewist buys a commercial pattern and picks a size from a static chart.
  2. They sew a test garment (muslin) to find fit problems.
  3. They manually pin, slash/spread, and redraw pattern pieces.
  4. They repeat until fit is acceptable, often across multiple rounds.

This process is slow, technical, and error-prone, which creates a high dropout point for new makers.

AI Decision Workflow

PatternPal reframes fitting as an AI-tailored decision workflow instead of trial-and-error.

  1. User selects a real pattern and enters measurements once.
  2. AI selects a starting size by comparing body measurements to pattern chart measurements.
  3. AI recommends where to alter and by how much (chest, waist, hip, sleeve, shoulder, length).
  4. UI renders plain instructions, visual highlights, and lookup terms for execution.
  5. User finalizes the plan by approving suggestions or entering custom muslin findings.

Human / AI Operating Boundary

AI Responsibilities

  1. Fit inference: choose base size and propose alteration magnitudes.
  2. Decision support: rank recommendations and provide confidence.
  3. Communication: convert fitting logic into beginner-safe guidance.

Human Responsibilities

  1. Provide and verify measurements.
  2. Approve, reject, or override each adjustment.
  3. Own final construction choices and garment outcome.

AI Responsibility Under Uncertainty

  1. Every recommendation is tied to explicit measurement mismatch signals.
  2. Confidence is surfaced so users can judge recommendation strength.
  3. Human approval is mandatory before any plan is treated as final.
  4. When certainty is weaker, the system biases toward smaller, safer edits.

Failure Modes And Mitigations

  1. Input errors: validation blocks impossible measurements.
  2. Outlier bodies or style mismatches: confidence drops and guidance becomes conservative.
  3. Model disagreement with real fitting: user can override and add custom adjustments.
  4. Decision traceability: approvals, rejections, and overrides are saved for audit and learning.

Learning Loop And Scale Path

  1. Per-user memory stores accepted, rejected, and custom adjustments.
  2. Outcome feedback (thumbs + muslin count) updates confidence behavior over time.
  3. Pattern-specific data can be expanded into trained fit models as labeled volume grows.
  4. Operational monitoring can track confidence drift and override rates at scale.

Outcome And Impact

  1. Reduces iteration time from fitting guesswork to guided first-pass adjustments.
  2. Lowers skill barriers by translating technical fitting into actionable language.
  3. Improves confidence and completion rates for new sewists.
  4. Supports more local, made-to-fit wardrobes and less fast-fashion dependence.
Prototype note: this MVP uses expert-authored heuristics plus synthetic historical fit data.

Next-Phase Features

The current MVP focuses on fit decision support. The next phase extends this into full pattern automation.

  1. Upload real commercial pattern files (PDF/DXF) directly into the app.
  2. Automatically map pattern piece geometry and detect editable adjustment zones.
  3. Apply AI-generated fit changes directly to pattern pieces, not just recommendations.
  4. Export print-ready altered patterns (A4/US Letter/tiled or large-format copy shop PDF).
  5. Generate a build sheet showing each change and where it was applied for traceability.
